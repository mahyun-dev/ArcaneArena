# Arcane Arena

## 1. 프로젝트 개요

Arcane Arena는 판타지 영웅과 지형 기믹을 활용해 최후의 1인이 살아남는 웹 기반 3D 배틀로얄 액션 게임이다. 최종 목표는 친구들과 가볍게 접속해서도 충분히 재미있고, 숙련되면 직업 상성과 지형 운영으로 실력을 드러낼 수 있는 밀도 높은 소규모 아레나를 만드는 것이다.

- 장르: 실시간 3D 배틀로얄 액션
- 플랫폼: Web, PC 우선 대응 후 모바일 조작 보조 지원
- 1차 목표: AI 대전 모드 완성
- 2차 목표: 실시간 멀티플레이 확장
- 핵심 기술: Three.js, Node.js, Socket.io, Cannon-es
- 배포 후보: GitHub Pages 프론트 배포 + 별도 서버, 또는 닷홈

## 2. 핵심 플레이 비전

이 게임의 재미는 단순히 스킬을 많이 맞히는 것이 아니라, 직업 고유 능력과 맵의 구조물을 함께 활용해서 전투 흐름을 바꾸는 데서 나온다.

핵심 경험은 다음 네 가지로 정의한다.

1. 전투를 시작하는 순간 직업별 전투 리듬이 확실히 달라야 한다.
2. 풀숲, 벽, 고지대, 자기장 같은 지형 요소가 승패에 직접 영향을 줘야 한다.
3. 난전에서도 읽을 수 있는 시각 효과와 명확한 피드백이 있어야 한다.
4. AI 모드만으로도 충분히 재미있는 전투 연습과 반복 플레이가 가능해야 한다.

## 3. 개발 우선순위

멀티플레이보다 먼저 AI 대전 모드를 완성한다. 이유는 전투의 손맛, 밸런스, 맵 구조, 이펙트, UI, 성능 최적화를 먼저 검증해야 이후 멀티플레이 전환 비용이 줄어들기 때문이다.

### 1차 개발 범위

- 플레이어 1명 + AI 봇 7명 대전
- 직업 8종 완성
- 메인 아레나 1종 완성
- 자기장 시스템 완성
- 기본 아이템과 상자 드롭 시스템 완성
- 인게임 HUD와 로비 UI 완성
- 로컬 저장 기반 설정, 전적 기록 완성

### 2차 개발 범위

- 실시간 룸 매칭
- 재접속 처리
- 관전 모드
- 서버 권한형 판정
- 동기화 보간 및 패킷 최적화

## 4. 게임 룰

- 기본 모드: 솔로 서바이벌
- 최소 인원: 2명 또는 AI 포함 8인 매치
- 최대 인원: 8명
- 승리 조건: 마지막까지 생존
- 평균 플레이 타임: 15분에서 30분
- 시작 배정: 매치 시작 시 모든 플레이어를 맵 내 유효 스폰 포인트에 무작위 배치
- 직업 배정: 매치 시작 시 모든 플레이어에게 8개 직업 중 하나를 무작위 부여
- 전투 시작: 게임 시작 즉시 이동 가능, 10초 뒤 중앙 주요 오브젝트 활성화
- 자기장 시작: 게임 시작 후 120초
- 자기장 축소 간격: 30초마다 축소
- 자기장 외곽 피해: 초당 최대 체력의 5% 고정 피해

## 5. 전투 시스템 설계

### 기본 전투 규칙

- 모든 직업은 기본 공격, 액티브 스킬, 패시브 스킬을 가진다.
- 자원은 직업별로 Mana 또는 Stamina를 사용한다.
- 기본 공격은 이동 중 사용 가능하되, 직업별로 공격 후 경직이 다르다.
- 스킬은 강력하지만 예측 가능해야 하므로, 시전 전조와 쿨타임을 명확히 보여준다.
- 군중 제어기는 연속으로 너무 오래 걸리지 않도록 저항 규칙을 둔다.

### 상태 이상 규칙

- 둔화: 이동속도 감소
- 속박: 이동 불가, 시점 전환과 공격은 가능
- 침묵: 스킬 사용 불가
- 은신: 적 미니맵에서 제거, 근거리에서는 희미한 실루엣 노출
- 화상: 짧은 지속 피해
- 약화: 받는 피해 증가

### 군중 제어 제한

- 같은 대상은 5초 안에 강한 이동 불가 효과를 두 번 이상 맞으면 두 번째부터 지속시간 50% 감소
- 무적 패시브 발동 후 8초 동안 추가 보호 패시브 재발동 불가

- 전사는 강하지만 원거리 견제에 취약해야 한다.
- 마법사는 폭딜이 강하지만 생존기가 약해야 한다.
- 암살자와 도적은 조작 난도가 높은 대신 성공 보상이 커야 한다.
- 사제는 생존형이지만 킬 캐치 능력은 낮아야 한다.
- 드루이드는 지형이 없는 평지에서 성능이 과도하지 않도록 제어 지속시간을 제한한다.
- 인간은 랜덤성이 있지만 기본 성능이 너무 낮아지지 않게 설계한다.

## 7. 월드와 맵 디자인

### 메인 맵 콘셉트

중세 판타지 원형 경기장을 중심으로, 외곽 폐허와 자연 지형이 섞인 입체형 전장을 구성한다. 중앙은 교전 유도가 강하고, 외곽은 생존과 매복에 유리한 구조로 만든다.

### 맵 레이어 구성

1. 중앙 결투장
2. 외곽 회랑
3. 고지 언덕 지역
4. 풀숲 매복 지역
5. 파괴 가능한 벽 구간
6. 전리품 상자 배치 구역

### 지형 오브젝트 규칙

#### 부서지는 벽

- 내구도 수치 보유
- 폭발, 돌진, 강한 근접 스킬에 파괴됨
- 파괴 시 새로운 시야와 이동 경로 생성
- 파편 연출은 시각 효과 위주로 처리하고 실제 물리 조각은 제한해서 성능 부담 축소

#### 풀숲

- 내부 캐릭터는 외부에서 완전히 보이지 않음
- 공격 또는 피격 시 1초 동안 위치 노출
- 드루이드는 추가 보너스를 받음

#### 고지대

- 원거리 공격 사거리 8% 증가
- 투사체 시야 확보에 유리
- 대신 엄폐물이 적어 노출 위험 증가

#### 마나 폭풍

- 푸른 마력 장벽이 외곽에서 수축
- 축소 전 예고 이펙트 표시
- 바깥 지역에 있는 플레이어는 지속 피해 + 화면 가장자리 왜곡 효과

## 8. 아이템 시스템

전투 리듬을 해치지 않는 범위에서 짧고 강한 선택지를 주는 방향으로 설계한다.

### 아이템 종류

- 회복 물약: 즉시 체력 25 회복
- 마나 수정: 마나 35 회복
- 강화 룬: 12초 동안 공격력 10% 증가
- 보호 룬: 8초 동안 받는 피해 15% 감소
- 재사용 촉매: 현재 스킬 쿨타임 20% 즉시 감소

### 드롭 규칙

- 상자는 맵 주요 포인트에 고정 생성
- 상위 등급 상자는 중앙과 고지대에 배치
- 인간은 상위 등급 드롭 확률 소폭 증가

## 9. AI 대전 모드 설계

AI 모드는 임시 콘텐츠가 아니라, 게임의 핵심 완성도를 검증하는 주력 모드로 설계한다.

### AI 목표

- 사람처럼 완벽하지는 않지만 전투 의도가 읽혀야 한다.
- 직업별 성격 차이가 느껴져야 한다.
- 맵 지형을 활용해야 한다.
- 자기장과 생존 판단을 할 수 있어야 한다.

### AI 구조

FSM만으로 시작하되, 각 상태에 가중치 기반 의사결정을 섞는다.

#### 기본 상태

1. 탐색: 안전 지역 안에서 상자 탐색, 중앙과 고지 우선 이동
2. 경계: 발소리, 피해 로그, 시야 단서 감지 시 주변 확인
3. 추격: 감지한 적에게 접근, 직업 사거리에 맞춰 거리 유지
4. 교전: 기본 공격, 스킬, 회피, 엄폐 사용
5. 후퇴: 체력 또는 자원이 낮을 때 시야를 끊고 도망
6. 재정비: 회복 아이템 사용, 유리한 지형 재점유
7. 자기장 대응: 안전 구역 우선 진입

### 직업별 AI 성격

- 전사: 가장 가까운 적에게 적극 돌진
- 마법사: 고지대와 중거리 유지 선호
- 암살자: 교전 중인 적의 측후방 진입 선호
- 도적: 짧은 진입과 이탈 반복
- 사제: 체력 낮을 때 성역 활용 후 재진입
- 드루이드: 풀숲 인접 경로를 우선 탐색
- 네크로멘서: 좁은 통로에서 교전 유도
- 인간: 상황별로 평균적인 판단 수행

### AI 난이도 단계

- 쉬움: 반응속도 느림, 스킬 연계 적음
- 보통: 자기장과 거리 조절 이해
- 어려움: 스킬 콤보, 측면 이동, 위험 회피 빈도 증가

## 10. UI/UX 설계

### 로비 화면

- 중앙에 현재 인원과 매치 상태 표시
- 좌측에 참가자 카드 목록과 준비 여부 표시
- 우측에 랜덤 직업 배정 규칙 안내와 능력 설명 도감 버튼 배치
- 하단에 시작 버튼과 설정 버튼 배치
- 배경은 회전하는 3D 경기장 프리뷰로 구성

### 인게임 HUD

- 좌측 하단: 초상화, HP, 자원 바, 상태 이상 아이콘
- 하단 중앙: 기본 공격, 스킬, 사용 아이템 슬롯, 쿨타임 숫자 및 원형 게이지
- 상단 중앙: 킬 로그, 자기장 경고, 라운드 알림
- 우측 상단: 미니맵, 안전 구역, 적 교전 핑, 주요 상자 위치
- 중앙 주변: 피격 방향 표시와 위험 경고

### UX 원칙

- 강한 스킬은 반드시 전조 연출과 사운드 구분이 있어야 한다.
- 난전에서 색상만으로 정보를 전달하지 않고 형태와 아이콘으로도 구분한다.
- 모바일 대응 시 자동 조준이 아니라 조작 보조 수준으로만 지원한다.

## 11. 아트 및 연출 방향

고퀄리티를 위해 단순히 오브젝트 수를 늘리기보다, 실루엣과 이펙트 가독성을 우선한다.

### 비주얼 키워드

- 어두운 석재 경기장 + 선명한 마법 광원
- 캐릭터별 전투 색상 고유화
- 원형 전장 중심의 강한 공간 인지성
- 과하지 않은 파티클, 읽기 쉬운 타격 이펙트

### 직업별 색상 가이드

- 네크로멘서: 청록, 어두운 회색
- 마법사: 주황, 금색
- 암살자: 자주빛 회색, 적색 포인트
- 전사: 강철색, 심홍
- 도적: 남색, 은색
- 사제: 백색, 청금색
- 드루이드: 녹색, 황토색
- 인간: 중립 회색, 청색 포인트

## 12. 기술 설계

### 클라이언트

- Three.js 기반 렌더링
- 캐릭터, 투사체, 이펙트, UI를 모듈 분리
- 애니메이션 상태머신 적용
- 모바일 대응은 반응형 HUD 레이아웃으로 처리

### 서버

- Node.js 기반 경기 상태 관리
- 멀티플레이 단계에서 서버 권한형 판정 적용
- 충돌, 피해, 스킬 명중, 자기장 판정은 서버가 최종 결정

### 물리

- Cannon-es 사용
- 플레이어, 벽, 투사체, 넉백 판정 처리
- 파괴 오브젝트는 단순화된 콜라이더 사용

### 네트워크 최적화

- 멀티플레이 전환 시 위치, 회전, 속도만 우선 동기화
- 중요 이벤트는 패킷 분리 전송
- 초기 버전은 JSON, 안정화 후 ArrayBuffer 구조로 이전 가능하게 설계

## 13. 성능 목표

- PC 기준 60 FPS 유지
- 중급 모바일 기기에서 30 FPS 이상 목표
- 한 화면 동시 파티클 수 제한
- 드로우콜 절감을 위한 모델 재사용 및 인스턴싱 고려
- 파괴 연출은 GPU 부담이 큰 물리 조각 대신 셰이더 및 짧은 프리셋 애니메이션 활용

## 14. 저장 데이터

## 18. 최종 방향 정리

Arcane Arena는 단순한 웹 데모가 아니라, 적은 인원으로도 긴장감 있는 전투가 가능한 완성형 소규모 배틀로얄을 목표로 한다. 따라서 처음부터 직업 개성, 지형 상호작용, 읽기 쉬운 이펙트, 안정적인 성능, AI 전투 재미를 핵심 품질 기준으로 잡고 개발한다.

우선은 AI 대전 모드에서 전투가 재미있는지부터 증명하고, 그 다음 멀티플레이를 올리는 순서가 가장 안전하다.
